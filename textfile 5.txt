Game Feature Requests — Consolidated Documentation

Source: Email thread between Clark Alden and Billy Koulax
Priority rule: If there is any conflict, Clark Alden’s requests take priority.

⸻

1. Core Gameplay Improvements (High Priority)

1.1 Driving & Car Physics Improvements

Improve the vehicle handling model to make driving feel smoother and more realistic.

Requested changes:
	•	Improve steering and directional control.
	•	Improve driving responsiveness.
	•	Improve drifting mechanics.
	•	Improve acceleration and deceleration behavior.
	•	Make it easier for cars to lose momentum naturally instead of maintaining speed unrealistically.

Goal:
	•	Driving should feel more responsive, predictable, and skill-based.

⸻

1.2 Ball Physics Improvements

Current ball behavior feels unrealistic.

Requested changes:
	•	Reduce the extremely fast initial movement of the ball.
	•	Prevent the ball from slowing down too quickly after moving fast.
	•	Adjust physics so the ball behaves in a more realistic and consistent way.

Goal:
	•	Ball motion should feel natural and balanced.

⸻

1.3 Goal Scoring Rules

Prevent unintended scoring mechanics.

Requested change:
	•	Players should not be able to score from behind the goal.

Goal:
	•	Ensure goals only count when the ball enters from the correct direction.

⸻

2. Gameplay Features

2.1 Boost Mechanics

Add additional boost mechanics.

Possible implementation:
	•	Introduce Super Boosts or stronger boost pickups.

Goal:
	•	Increase gameplay intensity and allow for high-skill plays.

⸻

2.2 Demolition System

Add a collision mechanic similar to Rocket League.

Requested feature:
	•	Cars should be able to destroy (demolish) other cars when hitting them at high speed.

Possible effect:
	•	Explosion effect (“kaboom”).
	•	Temporary respawn delay.

Goal:
	•	Add strategic aggression and excitement.

⸻

2.3 Replay System

Add a replay feature.

Requested functionality:
	•	Automatically replay goals or important moments.

Goal:
	•	Allow players to watch and enjoy big plays.

⸻

2.4 Box Score / Match Statistics

Add a scoreboard with performance stats.

Possible metrics:
	•	Goals
	•	Assists
	•	Saves
	•	Shots
	•	Demolitions

Goal:
	•	Provide performance feedback and competitive stats.

⸻

3. Environment & Map Mechanics

3.1 Climbable Walls

Add arena wall interaction.

Requested functionality:
	•	Cars can drive up walls.
	•	Players can jump off walls or the roof.
	•	Allows for trick shots.

Goal:
	•	Enable advanced aerial plays and trick shots.

Technical note:
	•	Implementation may be difficult due to 3D upside-down or sideways physics limitations in the current model.

⸻

3.2 Improved Render Distance

Improve visual performance.

Requested change:
	•	Increase render distance so objects appear earlier and arenas feel larger.

Goal:
	•	Improve visual clarity and immersion.

⸻

4. Game Modes

4.1 Online Multiplayer (Future Feature)

Add multiplayer once the game becomes popular.

Requested features:
	•	Online matchmaking
	•	Players can choose their teams

Goal:
	•	Enable competitive multiplayer gameplay.

Priority:
	•	Low priority / post-launch feature.

⸻

5. World / Level Design

5.1 Unique World Features (Clark Priority)

Each world should contain different mechanics or features.

Goal:
	•	Avoid worlds feeling identical.
	•	Encourage players to adapt strategies depending on the map.

⸻

6. Developer Mode Improvements (Clark Priority)

Improve the developer mode settings so they make sense for advanced users.

Possible changes:
	•	Clear dev tools.
	•	Logical developer settings.
	•	Useful debugging controls.

Goal:
	•	Make development and testing easier and more intuitive.
